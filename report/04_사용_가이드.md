## 시작하기

### 1. 시스템 요구사항

#### 필수 요구사항
- **웹 브라우저**: Chrome, Firefox, Safari, Edge (최신 버전)
- **Python 3.x** (서버 실행용, 선택사항)
- 또는 **Node.js** (http-server 사용 시)

#### 권장 사양
- 모던 웹 브라우저
- 인터넷 연결 (초기 로드 시)

---

### 2. 설치 및 실행

#### 방법 1: Python HTTP Server 사용

**Windows:**
```bash
# 배치 파일 실행
start_server.bat

# 또는 직접 실행
python -m http.server 8000
```

**Linux/Mac:**
```bash
# 쉘 스크립트 실행
chmod +x start_server.sh
./start_server.sh

# 또는 직접 실행
python3 -m http.server 8000
```

#### 방법 2: Node.js http-server 사용
```bash
# http-server 설치 (최초 1회)
npm install -g http-server

# 서버 실행
http-server -p 8000
```

#### 방법 3: 직접 파일 열기
- `index.html` 파일을 브라우저로 직접 열기
- ⚠️ 주의: 로컬 스토리지 기능은 일부 브라우저에서 제한될 수 있음

---

### 3. 브라우저 접속

서버 실행 후 브라우저에서 다음 주소로 접속:
```
http://localhost:8000
```

## 기본 사용법

### 탭 네비게이션

화면 상단의 탭을 클릭하여 원하는 섹션으로 이동할 수 있습니다.

#### 탭 목록
1. **개요**: 프로젝트 소개 및 기본 원칙
2. **부서원 관리**: 부서원 등록 및 관리
3. **연차별 역량**: L1~L5 역량 체크리스트
4. **성향 트랙**: 엔지니어 성향 소개

6. **트랙 비교**: 전문가 vs 관리 트랙
7. **평가 기준**: 평가 구조 및 승진 기준

## 부서원 관리

### 1. 부서원 등록

#### 단계별 가이드

1. **"부서원 관리" 탭 클릭**
   - 상단 네비게이션에서 "부서원 관리" 탭 선택

2. **정보 입력**
   - **이름**: 부서원의 이름 입력 (필수)
   - **주요 성향**: 5가지 성향 중 선택 (필수)
   - **보조 성향**: 선택사항 (없으면 "없음" 선택)
   - **근무 연차**: 숫자로 입력 (0~50년, 필수)
   - **업무 레벨**: L1~L5 중 선택 (필수)

3. **등록 버튼 클릭**
   - 모든 필수 항목 입력 후 "등록" 버튼 클릭
   - 등록된 부서원이 목록에 자동으로 표시됨

#### 입력 예시
```
이름: 홍길동
주요 성향: 🔧 문제 해결형
보조 성향: 🏗️ 설계/아키텍처형
근무 연차: 5
업무 레벨: L2 - Practitioner (4~7년)
```

---

### 2. 부서원 목록 조회

#### 표시 정보
- **이름**: 부서원 이름
- **업무 레벨**: L1~L5 배지
- **근무 연차**: 연차 배지
- **주요 성향**: 성향 배지
- **보조 성향**: 성향 배지 (있는 경우)

#### 정렬 기준
- 레벨 우선 (L5 → L1)
- 동일 레벨 내 연차 내림차순

---

### 3. 부서원 정보 수정

#### 수정 방법

1. **목록에서 "수정" 버튼 클릭**
   - 수정할 부서원 카드의 "수정" 버튼 클릭

2. **폼에 정보 자동 입력**
   - 기존 정보가 폼에 자동으로 채워짐

3. **정보 수정**
   - 필요한 항목만 수정

4. **"등록" 버튼 클릭**
   - 수정된 정보 저장

---

### 4. 부서원 삭제

#### 삭제 방법

1. **목록에서 "삭제" 버튼 클릭**
   - 삭제할 부서원 카드의 "삭제" 버튼 클릭

2. **확인 대화상자**
   - "정말 삭제하시겠습니까?" 확인 메시지 표시

3. **확인 클릭**
   - 확인 시 삭제 완료
   - 취소 시 삭제 취소

⚠️ **주의**: 삭제된 데이터는 복구할 수 없습니다.

---

### 5. 데이터 저장

#### 저장 위치
- **로컬 스토리지**: 브라우저에 저장
- 브라우저별로 독립적으로 저장됨
- 페이지를 새로고침해도 데이터 유지

#### 데이터 백업
현재 버전에서는 자동 백업 기능이 없습니다.
- 브라우저 개발자 도구에서 수동으로 데이터 확인 가능
- 향후 버전에서 내보내기 기능 추가 예정

## 정보 조회

### 1. 연차별 역량 조회

#### 사용 방법
1. **"연차별 역량" 탭 클릭**
2. **레벨 선택**: L1~L5 중 관심 있는 레벨 확인
3. **체크리스트 확인**: 각 역량 항목 확인

#### 활용 방법
- 개인 역량 평가 시 체크리스트 활용
- 성장 목표 설정 시 참고
- 1:1 면담 시 기준으로 활용

---

### 2. 성향 트랙 조회

#### 사용 방법
1. **"성향 트랙" 탭 클릭**
2. **성향 카드 확인**: 5가지 성향 소개 확인
3. **Primary/Secondary 선택**: 부서원 등록 시 참고

#### 활용 방법
- 부서원의 성향 파악
- 팀 구성 시 참고
- 멘토링 매칭 시 활용

---

### 3. 면담 가이드 활용

#### 사용 방법
1. **"면담 가이드" 탭 클릭**
2. **레벨별 질문 확인**: 해당 레벨의 질문 예시 확인
3. **면담 시 활용**: 실제 1:1 면담 시 질문 참고

#### 활용 팁
- 질문을 그대로 사용하지 말고 상황에 맞게 변형
- 추가 질문을 자유롭게 추가
- 면담 후 피드백 기록 권장

---

### 4. 트랙 비교

#### 사용 방법
1. **"트랙 비교" 탭 클릭**
2. **전문가 트랙 vs 관리 트랙 비교**
3. **선택 가이드 확인**: 트랙 선택 시 참고

#### 활용 방법
- 부서원의 커리어 경로 상담 시 활용
- 트랙 전환 시 기준으로 활용

---

### 5. 평가 기준 확인

#### 사용 방법
1. **"평가 기준" 탭 클릭**
2. **평가 구조 확인**: 항목별 비중 확인
3. **승진 기준 확인**: 승진 판단 기준 확인

#### 활용 방법
- 평가 시 기준으로 활용
- 승진 심사 시 참고
- 성장 목표 설정 시 활용

## FAQ

### Q1. 데이터가 사라졌어요
**A**: 로컬 스토리지는 브라우저별로 저장됩니다. 다른 브라우저나 시크릿 모드에서는 데이터가 보이지 않을 수 있습니다. 또한 브라우저 데이터를 삭제하면 저장된 정보도 함께 삭제됩니다.

### Q2. 여러 명이 동시에 사용할 수 있나요?
**A**: 현재 버전은 로컬 스토리지를 사용하므로 각 사용자의 브라우저에 독립적으로 저장됩니다. 공유하려면 데이터를 내보내기/가져오기 기능이 필요합니다 (향후 추가 예정).

### Q3. 모바일에서 사용할 수 있나요?
**A**: 네, 반응형 디자인으로 모바일에서도 사용 가능합니다. 다만 서버를 실행한 후 모바일 브라우저에서 접속해야 합니다.

### Q4. 데이터를 백업할 수 있나요?
**A**: 현재 버전에서는 자동 백업 기능이 없습니다. 브라우저 개발자 도구(F12) → Application → Local Storage에서 수동으로 확인할 수 있습니다.

### Q5. 부서원이 많아지면 느려지나요?
**A**: 로컬 스토리지를 사용하므로 일반적으로 수백 명까지는 문제없이 동작합니다. 매우 많은 데이터의 경우 검색/필터 기능 추가가 필요할 수 있습니다.

## 문제 해결

### 문제 1: 서버가 실행되지 않아요

#### 해결 방법
1. **Python 설치 확인**
   ```bash
   python --version
   ```
   Python 3.x가 설치되어 있는지 확인

2. **포트 충돌 확인**
   - 8000번 포트가 사용 중인지 확인
   - 다른 포트 사용: `python -m http.server 8080`

3. **방화벽 확인**
   - 방화벽이 포트를 차단하지 않는지 확인

---

### 문제 2: 데이터가 저장되지 않아요

#### 해결 방법
1. **브라우저 설정 확인**
   - 쿠키 및 사이트 데이터 허용 확인
   - 시크릿 모드에서는 로컬 스토리지가 제한될 수 있음

2. **브라우저 개발자 도구 확인**
   - F12 → Console에서 에러 메시지 확인
   - Application → Local Storage에서 데이터 확인

---

### 문제 3: 스타일이 깨져요

#### 해결 방법
1. **브라우저 캐시 삭제**
   - Ctrl + F5 (강제 새로고침)

2. **브라우저 호환성 확인**
   - 모던 브라우저 사용 권장
   - Internet Explorer는 지원하지 않음

---

### 문제 4: 모바일에서 레이아웃이 이상해요

#### 해결 방법
1. **뷰포트 설정 확인**
   - 모바일 브라우저에서 정상 작동해야 함

2. **화면 회전**
   - 가로/세로 모드 모두 지원
