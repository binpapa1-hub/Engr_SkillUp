# Engineering Capability Growth Framework (ECGF)
## 기능 명세서

---

## 1. 부서원 관리 시스템

### 1.1 부서원 등록

#### 기능 설명
부서원의 기본 정보와 기술 역량 정보를 등록합니다.

#### 입력 항목
| 항목 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 이름 | Text | ✅ | 부서원 이름 |
| 주요 성향 (Primary) | Select | ✅ | 5가지 성향 중 선택 |
| 보조 성향 (Secondary) | Select | ❌ | 5가지 성향 중 선택 (선택사항) |
| 근무 연차 | Number | ✅ | 0~50년 범위 |
| 업무 레벨 | Select | ✅ | L1~L5 중 선택 |

#### 성향 옵션
1. 🔧 문제 해결형
2. 🏗️ 설계/아키텍처형
3. 🔬 연구/개선형
4. ⚙️ 현장/운영형
5. 👥 리더/멘토형

#### 레벨 옵션
- **L1**: Foundation (1~3년)
- **L2**: Practitioner (4~7년)
- **L3**: Senior Engineer (8~12년)
- **L4**: Staff / Principal (13~20년)
- **L5**: Master (20년+)

#### 검증 규칙
- 이름은 필수 입력
- 주요 성향은 필수 선택
- 근무 연차는 0 이상의 정수
- 업무 레벨은 필수 선택

---

### 1.2 부서원 목록 조회

#### 기능 설명
등록된 모든 부서원을 목록으로 표시합니다.

#### 표시 정보
- 이름
- 업무 레벨 (배지 형태)
- 근무 연차 (배지 형태)
- 주요 성향 (배지 형태)
- 보조 성향 (있는 경우, 배지 형태)

#### 정렬 기준
1. 업무 레벨 (L5 → L1 순)
2. 근무 연차 (내림차순)

#### 레이아웃
- 카드 기반 레이아웃
- 반응형 그리드 (모바일 대응)

---

### 1.3 부서원 정보 수정

#### 기능 설명
등록된 부서원의 정보를 수정합니다.

#### 동작 방식
1. 목록에서 "수정" 버튼 클릭
2. 폼에 기존 정보 자동 입력
3. 수정 후 "등록" 버튼 클릭
4. 변경사항 저장

#### 수정 가능 항목
- 모든 입력 항목 수정 가능

---

### 1.4 부서원 삭제

#### 기능 설명
등록된 부서원을 삭제합니다.

#### 동작 방식
1. 목록에서 "삭제" 버튼 클릭
2. 확인 대화상자 표시
3. 확인 시 삭제 완료

#### 주의사항
- 삭제된 데이터는 복구 불가
- 확인 대화상자로 실수 방지

---

### 1.5 데이터 저장

#### 저장 방식
- **로컬 스토리지 (LocalStorage)**
- 브라우저별로 독립적으로 저장
- 페이지 새로고침 후에도 데이터 유지

#### 데이터 구조
```json
{
  "id": "고유ID",
  "name": "이름",
  "primaryArchetype": "주요 성향",
  "secondaryArchetype": "보조 성향",
  "years": 연차,
  "level": "레벨"
}
```

## 2. 정보 조회 시스템

### 2.1 개요 탭

#### 표시 내용
- 프로젝트 목적
- 기본 원칙
- 성장 방법 (4가지)
- 성공 지표
- 최종 목표

---

### 2.2 연차별 역량 탭

#### 표시 내용
- L1~L5 각 레벨별 상세 역량 체크리스트
- 기술 역량
- 문제 해결
- 협업
- 기술 자산화

#### 레벨별 상세 정보
각 레벨별로 다음 정보를 제공:
- 연차 범위
- 역할 설명
- 역량 체크리스트 (카테고리별)

---

### 2.3 성향 트랙 탭

#### 표시 내용
- 5가지 엔지니어 성향 소개
- 각 성향별 설명
- Primary/Secondary 개념 설명

#### 성향 카드
각 성향을 카드 형태로 표시:
- 성향 이름 (이모지 포함)
- 성향 설명

---

### 2.4 면담 가이드 탭

#### 표시 내용
- 공통 면담 구조 (5단계)
- 레벨별 질문 예시
  - L1 질문 (3개)
  - L2 질문 (3개)
  - L3 질문 (3개)
  - L4~L5 질문 (3개)

---

### 2.5 트랙 비교 탭

#### 표시 내용
- 전문가 트랙 (IC)
  - 레벨별 역할
  - 평가 기준
- 관리 트랙 (Manager)
  - 레벨별 역할
  - 평가 기준
- 선택 가이드

---

### 2.6 평가 기준 탭

#### 표시 내용
- 평가 구조 (항목별 비중)
- 승진 판단 기준
- 관리자 체크 질문

---

## 3. 사용자 인터페이스

#### 탭 구조

3. 연차별 역량
